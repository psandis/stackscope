# Vision

Stackscope should let an architect point at a repo, scan what is there, and draw an understandable ecosystem view within minutes.

## Product Principles

- Prefer fast inference over perfect completeness.
- Keep the internal model small and explainable.
- Make drawing a first-class outcome, not a side effect.
- Derive multiple outputs from one model.
- Treat source artifacts as evidence, not ground truth.
- Stay CLI-first in V1 so the workflow fits existing engineering environments.

## User Value

- understand what exists
- understand what depends on what
- get a picture of the ecosystem without drawing it manually
- identify likely data stores and external integrations
- see delivery and infrastructure signals quickly
- generate usable documentation without hand-drawing architecture diagrams

## Long-Term Direction

Over time, Stackscope can evolve from repo scanning into a broader blueprinting engine that merges evidence from source control, CI/CD, IaC, deployment manifests, and runtime metadata.
